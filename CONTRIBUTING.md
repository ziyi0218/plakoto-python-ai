# Contributing

Thank you for your interest in contributing to this project.

## Ways to Contribute

You can contribute by:

- reporting bugs
- suggesting improvements
- improving documentation
- refactoring code
- adding tests

## Before Contributing

Please:

1. read the project documentation
2. check whether a similar issue already exists
3. keep your changes focused and well explained

## Development Guidelines

- use clear and readable Python code
- preserve the current project structure when possible
- avoid unrelated large refactors in the same change
- add comments or docstrings when needed
- test your changes before submitting

## Pull Requests

When opening a pull request, please include:

- a short summary of the change
- the reason for the change
- any relevant screenshots or terminal output if applicable

## Issues

When opening an issue, please provide:

- a clear title
- steps to reproduce the problem
- expected behavior
- actual behavior
- environment details if relevant

## Scope

This repository was originally created as a university project. Contributions should remain consistent with its educational and technical goals.
